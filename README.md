# デジタル軍議システム (Persona-based Discussion Simulator)

Ollama（ローカルLLM）を活用し、複数の異なるペルソナを持つAIエージェント間で多角的な議論を行わせるシミュレーションアプリです。

## 🌟 プロジェクト概要
このシステムは、一つの議題に対して「厳格な決裁者」「品質重視の技術者」など、性格の異なる複数のAIを参加させ、あえて「反論」や「懸念」をぶつけ合わせることで、人間の意思決定をサポートするための「デジタル会議」の場を提供します。

## 🛠 技術スタック
- **Python 3.10+**
- **Streamlit**: インタラクティブなUIの構築
- **Ollama**: ローカル環境でのLLM実行基盤
- **Gemma2:9b**: 推論に使用したメインモデル

## 📝 主な機能
- **動的ペルソナ選択**: 事前に設定された5種類の役職から参加者を選択可能。
- **カスタムエージェント作成**: ユーザーが独自の性格設定を持つエージェントを即座に追加可能。
- **会議録（Transcript）形式のコンテキスト保持**: 議論の全履歴をAIに読み込ませることで、文脈を汲み取った反論を実現。
- **自己修正（Retry）ロジック**: AIの発言が短すぎたり、安易に同調したりした場合に、自動で最大3回まで再生成を促す品質管理機能。

## 🚀 実行方法
1. Ollamaを起動し、モデルをプルする
```bash
ollama pull gemma2:9b
```
2. アプリを起動する
```bash
streamlit run app.py
```

## 🔍 技術検証結果（PoCレポート）
本プロジェクトの開発を通じ、ローカルLLM（Ollama）を用いた多角議論の可能性を検証しました。その結果、以下の知見を得ました。

1. **推論能力の限界**: 
   複雑な多階層の議論において、ローカルLLM（9bクラス）では文脈の長期保持や、高度な論理的矛盾の指摘、および「同調圧力に屈しない独立した思考」の維持に限界があることが確認されました。
2. **クラウドLLMへの移行判断**: 
   より実用的かつ精度の高い議論・分析を実現するためには、より大規模なパラメータを持つクラウド型LLM（OpenAI/Claude等）の活用が不可欠であると判断しました。
